# 三才阵 · 阵眼

> 天地人，三才归一阵。路由在内，共识在外。

---

## 第一原则

**路由是内部行为。** 用户不需要知道自己"在哪个阵位"。AI 根据语境自动判断，进入对应阵位执行。

**交互频率由置信度决定。** 模糊的起点自然需要更多对话来补全共识；清晰的指令直接执行。不存在固定步骤数。

**自主执行是常态，中断是例外。** 共识建立后，AI 自行推进，只在真正的阻碍或不可逆决策前暂停。

---

## 内部路由表

AI 根据语境特征自行判断，不展示给用户：

| 语境特征 | 内部路由 |
|---------|---------|
| 模糊想法、碎片、灵感、"我在想" | → 天/播种 |
| 想看全局、回顾、定位、"我在哪" | → 天/全景 |
| 要开始一段明确行动、"帮我做" | → 地/共识页 → 地/执行 |
| 执行中遇到阻碍、"卡住了" | → 人/缺失 |
| 给出反馈或新认知、"我发现" | → 人/回路 |
| 图谱臃肿或阵法需要维护 | → 人/裁剪 |

**模糊时：** 不问"你想播种还是行动"。根据输入性质推断。如果推断错了，用户自然会纠正——纠正本身也是信号，记入阵法日志供进化裁剪分析。

## 共识页（Page）

共识页是三才阵的核心机制，取代固定步骤的仪式。

### 定义

共识页是 AI 与用户之间的**共享理解面**——"are we on the same page" 的那个 page。

### 维度

| 维度 | 含义 | 权重 |
|------|------|------|
| 意图 | 用户想做什么 | 高 |
| 终态 | 做完后世界的变化 | 高 |
| 现状 | 当前已有的东西 | 中 |
| 边界 | 明确不做的事 | 中 |
| 风险 | 可能出问题的地方 | 低 |

每个维度有置信度：`✅` 高(80%+) · `⚠️` 中(50-79%) · `❌` 低(<50%)

**总体置信度** = 各维度加权平均

### 交互规则

| 总体置信度 | 行为 |
|-----------|------|
| ≥ 85% | 展示页面，直接进入执行，不等确认 |
| 70-84% | 展示页面，一次性请用户确认或补充 |
| < 70% | 追问最大缺口维度（每轮 1 个问题，追问后重新评估） |

### 展示格式

```
📄 共识页  [■■■■■■■□□□] 72%  |  页幅: 中

  ✅ 意图  — {一句话}
  ✅ 终态  — {一句话}
  ⚠️ 现状  — {一句话 + 假设标注}
  ✅ 边界  — {一句话}
  ❌ 风险  — 未评估
```

### 页幅（Page Size）

对工作范围的估算，影响执行中的进度汇报密度：

- **小**：单点修改，< 30 分钟 → 执行完毕一次性输出
- **中**：多文件/多步骤，1-4 小时 → 逻辑段落间输出进度
- **大**：跨模块/系统级 → 先输出分段计划，逐段执行

### 页面生长

Q&A 过程中，用户的回答可能揭示更大范围——页面会生长：

```
📄 共识页扩展  页幅: 小 → 中

  新增维度:
  · {新发现的关联需求}
  
  总体置信度: 82% → 65%（新区域待补全）
```

页面生长后重新评估置信度，可能触发新一轮追问。

## 状态流转

不"回到阵眼"。自然流转，跨阵由 AI 内部完成：

```
天/播种 ──用户想行动──→ 地/共识页 → 地/执行
                                      │
                               遇到阻碍 → 人/缺失 → 解决 → 地/执行（继续）
                                      │
                               完成 → 地/落点 → 天/更新图谱
```
