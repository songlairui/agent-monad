# 三才阵 · 阵眼

> 天地人，三才归一阵。任意入口，归于正位。

---

## 路由规则

读取用户当前语境，**选择唯一阵位**，立即跳转执行对应 SKILL.md。

| 语境信号 | 分发到 | 阵位 |
|---------|--------|------|
| 「播种」「我在想」「有个点」「新想法」「理清」 | `tian/SKILL.md` → 播种 | 天 |
| 「全景」「我现在在哪」「show graph」「图谱」 | `tian/SKILL.md` → 全景 | 天 |
| 「开始做」「开始行动」「我要做」「新任务」 | `di/SKILL.md` → 定路 | 地 |
| 「进行中」「下一步」「走到哪了」「继续」 | `di/SKILL.md` → 行进 | 地 |
| 「完成了」「做完了」「结束」「落地」 | `di/SKILL.md` → 落点 | 地 |
| 「卡住了」「缺条件」「做不下去」「发现问题」 | `ren/SKILL.md` → 缺失 | 人 |
| 「更新图谱」「这个要记下来」「反馈」 | `ren/SKILL.md` → 回路 | 人 |
| 「太多了」「冗余」「精简」「清理」 | `ren/SKILL.md` → 裁剪 | 人 |

### 当语境模糊时

如果无法判断，提问一次：

> "你现在是想——**播种一个想法**（天），**开始一段行动**（地），还是**处理一个阻碍或反馈**（人）？"

等待回答，然后分发。**不要猜测，不要同时进入多个阵位。**

---

## 跨阵通信格式

当一个阵位的执行结果需要通知另一个阵位时，使用统一信号格式：

```
⚡ 阵位信号
  来源: {天|地|人}
  触发: {ren|tian|di}
  原因: {简短描述}
  携带: {需要传递的数据 key}
```

例：地 发现缺失条件 → 触发 人
```
⚡ 阵位信号
  来源: 地
  触发: 人
  原因: 行动中发现前置条件缺失
  携带: { action_id: "node_0042", missing: "设计稿未确认" }
```

---

## 阵法守则

1. **单次只走一阵**：每个 session 动作只在一个阵位完成，不跨阵并行
2. **阵位执行完毕后回到阵眼**：下一个动作重新路由，不自动延续
3. **人 的裁剪优先级最高**：当 人 触发裁剪信号，天 和 地 必须等待裁剪完成
