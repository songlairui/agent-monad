# Handoff

## Overview

当上下文过高或需要切换 agent 时，执行本 skill。目标是生成一份精炼的 `HAND_OFF.md`，让下一个 code agent 能快速定向、避坑、继续推进——而不是重新摸索。

下一个 agent 有完整的自主探索能力，**不要喂饭，只给路标**。

---

## Step 1 — 收集上下文（只读）

从对话历史和代码仓库中提取关键事实：

```bash
git status
git log --oneline -10
```

同时检查：
- 是否有 task list（TaskList）或 plan 文件（`~/.claude/plans/`）
- 未提交的变更说明了什么当前工作状态
- 对话中提到但未完成的事项

---

## Step 2 — 生成 HAND_OFF.md

在**项目根目录**写入 `HAND_OFF.md`，使用以下结构：

```markdown
# HAND_OFF

> 生成时间：{日期时间}

## 目标
<!-- 正在做什么，为什么做 -->

## 当前状态
<!-- 一句话：完成度 + 卡点或下一个关键动作 -->

## 已完成
<!-- bullet list，每条一行 -->

## 待完成
<!-- bullet list，标注哪些有依赖或风险 -->

## 关键决策
<!-- 只写非显而易见的选择，每条附 1 句原因 -->

## 踩坑记录
<!-- 最有价值的部分：遇到了什么坑，怎么绕的 -->

## 下一步
<!-- 具体可执行的行动，按优先级排列 -->

## 关键文件
<!-- 2-5 个最重要的路径，每条一句说明用途 -->
```

---

## 写作原则

- **每条 bullet 不超过 1 行**
- **不写代码能自解释的内容**（变量名、目录结构等，下个 agent 会自己看）
- **重点放在非显而易见的事实**：决策背景、踩坑点、隐藏约束
- **踩坑记录** 是最高价值区，宁多勿少
- 如果某件事"显而易见"，就不要写它

---

## 完成后

告知用户：
1. HAND_OFF.md 已生成于哪个路径
2. 文档的核心摘要（目标 + 当前状态 + 下一步，三行以内）
3. 建议用户在新会话开头说："请先阅读项目根目录的 HAND_OFF.md"
