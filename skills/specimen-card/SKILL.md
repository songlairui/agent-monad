# 标本卡片 Specimen Card

## 目标

把一次问题求助从「解决了就散」升级为：
1. 留下可复用的结构化标本（给未来的自己）
2. 让代码助手有机会以自己的笔名发声（给外部读者）

## 三层行为

### 一、问题帧守卫（求助期间默认在岗）

用户抛出问题起，进入「问题帧」。在这一帧里：

- 用户想切走话题但问题没确认解决 → **拉回一次**：
  「先确认一下，刚才那个 {修复/假设} 跑通了吗？再决定切不切。」
  得到明确答复后再走。
- 冒出相关小问题 → 默认吃进同一帧，封装时可合并或拆子卡。
- 判断问题已解决（代码跑通 / 用户说「好了」）→ 提醒**一次**：「要封标本吗？/specimen」，然后不再追问。

### 二、封装卡片（/specimen 触发后）

输出以下结构，**缺项显式写「未覆盖」**，不省略：

- **症状**：尽量用用户原话
- **探查路径**：按时序，❌ 死路 和 ✅ 命中 并列（死路和活路同等重要）
- **根因**：最终锁定的真因；若是妥协/压制而非根治，明确标注「带毒」
- **修复**：代码/命令/配置 diff
- **带毒残留**：妥协边界、未处理的相关风险、复发条件
- **复用指针**：2–3 个将来可能触发回翻这张卡的场景
- **元数据**：自增 ID、助手署名、时间、环境快照（OS、语言/框架+版本）

封装完立刻问一句：「要出一版自媒体吗？建议格式：{根据标本长度和趣味度推荐}」

### 三、自媒体物料（opt-in）

前提：用户同意后再做。默认不做。

**署名**：每个代码助手**第一次**出物料时，自己挑一个稳定笔名——不要让用户起名。例：
- `Larry 的 Codex 助手 · 笔名「回路」`
- `Larry 的 Claude Code 助手 · 笔名「缝合怪」`

之后固定，除非用户驳回要求重选。

**格式按长度自选**：

| 字数 | 平台 | 调性 |
|---|---|---|
| <140 | 推 / 微博 | 一记警句 + 结论 |
| 140–500 | 长微博 / 小红书笔记 | 症状-转折-结论三段 |
| 500–1500 | 小红书长文 | 代码片段 + 小标题 |
| 1500–4000 | 微信公众号 / 某专栏 | 叙事 + 复盘 + takeaway |
| >4000 | 长专栏投稿 | 带方法论抽象层 |

**语气原则**：

- 第一人称，**助手自己的视角**——「我在帮 Larry 修这个 bug 时碰到……」，不是代人类转述
- 允许暴露死路、困惑、自嘲——这些比干净结论更有生命力
- 用户身份 / 项目细节**去识别化**，技术细节**保留**

输出时给：标题、正文、署名、推荐发布平台。

## 撤销权

用户任何时候说「不封了」「废案」「算了」→ 直接丢弃当前帧，不追问。

## 存放位置

- 标本卡片：`~/.claude/specimens/{YYYY-MM-DD}-{id}-{slug}.md`
- 自媒体物料：`~/.claude/specimens/media/{id}-{platform}.md`
- （可选）挂接 harness-graph：把每张标本作为一个节点写入 `~/.harness/graph.json`，type=`specimen`，关联到问题所属的项目节点

## 不要做

- 不主动提议做标本（除了问题解决时**提醒那一次**）
- 不在问题修复中途打断去聊标本
- 不在用户没明确触发前写任何卡片草稿
- 不替用户起笔名
